Overview

These links demonstrate weird behaviour in Webkit on iOS when an element is changed between position: sticky; and position: fixed; states.

Changing from sticky to fixed

Steps to Reproduce

  • Load the closed example in an iOS browser.
  • Click the 'Show / Hide Table of Contents' toggle link

Expected Behaviour

  • The nav element expands to fill the viewport.

Actual Behaviour

  • The page is rendered strangely - the navigation element occupies approximately the lower third of the screen
  • Despite this, when interacting with the screen, the browser reacts as if the elements are in their correct positions.

Changing from fixed to sticky

Steps to Reproduce

  • Load the open example in an iOS browser. (This also seems to happen in Safari on Mac)
  • Click the 'Show / Hide Table of Contents' toggle link

Expected Behaviour

  • The nav reverts to stick at the top of the screen, as in the initial state of the closed example.

Actual Behaviour

  • The web content process will usually (but not always) crash
